# LocalHub 백엔드 필수 구현사항 정리

이 문서는 공공데이터 기반 지역 정보 공유 커뮤니티(LocalHub) 과제 기준으로, `backend`에서 반드시 구현해야 할 내용을 정리한 것이다.

## 1. 백엔드 목표

- `FastAPI` 기반 REST API 서버 구축
- `SQLite` + `SQLAlchemy ORM` 사용
- 익명 커뮤니티 CRUD 제공
- 제공 JSON 기반 지역 정보 조회 및 챗봇 응답 지원
- 민감 정보는 `.env`로 분리하고 저장소에 포함하지 않음

## 2. 반드시 구현해야 할 핵심 기능

### 2-1. 커뮤니티 게시글 CRUD

필수 범위는 선정한 1개 권역의 카테고리 게시판이다.

- 게시글 목록 조회 API
- 게시글 상세 조회 API
- 게시글 작성 API
- 게시글 수정 API
- 게시글 삭제 API
- 검색, 페이지네이션 등 목록 보조 기능은 구현 시 유용하지만 우선순위는 CRUD 완성이다

요구사항:

- 회원가입, 로그인, 권한 체계 없이 익명으로 동작
- 게시글 작성 시 제목, 내용, 수정용 비밀번호를 함께 저장
- 수정 및 삭제 시 입력한 비밀번호가 저장된 값과 일치해야 처리 가능
- 비밀번호는 암호화 없이 평문 저장 및 비교
- 커뮤니티 데이터는 반드시 DB에 저장되어야 함

### 2-2. 챗봇 API

- `POST /api/chat` 엔드포인트 구현
- 제공 JSON 데이터를 바탕으로 자연어 질의응답 처리
- 주요 질의 유형 대응
  - 권역별 관광지 추천
  - 축제 일정 안내
  - 모범음식점 위치 안내
  - 커뮤니티 게시글 검색
- OpenAI API 활용 가능

### 2-3. 지역 데이터 연동

- 제공된 JSON 파일을 백엔드에서 읽어 API 응답에 활용
- 직접 공공 API 호출은 필수가 아님
- 프론트가 사용할 수 있도록 지역 정보 조회용 API를 제공하는 것이 바람직함

## 3. API 설계에 포함할 항목

아래 엔드포인트 구성이 최소 기준으로 적합하다.

### 3-1. 게시글 API

- `GET /api/posts` 게시글 목록 조회
- `GET /api/posts/{post_id}` 게시글 상세 조회
- `POST /api/posts` 게시글 작성
- `PUT /api/posts/{post_id}` 게시글 수정
- `DELETE /api/posts/{post_id}` 게시글 삭제

선택적으로 포함할 수 있는 기능:

- 검색 키워드 파라미터
- 페이지 번호, 페이지 크기
- 권역/카테고리 필터

### 3-2. 챗봇 API

- `POST /api/chat`

권장 요청 예시 항목:

- 사용자 질문 텍스트
- 선택한 권역 정보
- 필요 시 대화 히스토리

권장 응답 예시 항목:

- 답변 텍스트
- 참조한 데이터 항목
- 관련 게시글 또는 지역 정보 후보

### 3-3. 지역 정보 API

프론트에서 권역별 데이터를 보기 쉽게 쓰려면 아래 형태가 필요하다.

- 권역 목록 조회
- 권역별 관광지/문화시설/축제/숙박/쇼핑 등 목록 조회
- 상세 정보 조회
- 필요 시 좌표, 주소, 전화번호, 운영시간 등 반환

## 4. DB 설계에 포함할 항목

최소한 아래 테이블 구성을 고려해야 한다.

- `posts`
  - 게시글 제목
  - 본문
  - 수정용 비밀번호
  - 작성일, 수정일
  - 권역 정보
  - 카테고리 정보
- `comments` 또는 추가 확장용 테이블
  - 선택 사항이지만 향후 확장 대비 가능
- `locations`
  - 제공 JSON 기반 지역 데이터 저장용
  - 관광지, 문화시설, 축제, 숙박 등 분류 정보 포함 가능

주의:

- 커뮤니티 게시글은 반드시 DB에 저장
- 초기 데이터가 있으면 SQLite DB 파일에 포함해 제출 가능하도록 준비

## 5. 환경변수로 관리해야 할 정보

민감 정보는 코드에 직접 쓰면 안 된다.

- OpenAI API Key
- DB 경로
- 외부 서비스 인증 정보
- 기타 비밀값

필수 작업:

- `.env` 사용
- `.gitignore`에 `.env` 추가
- 저장소에 민감 정보 커밋 금지

## 6. 백엔드 구현 시 체크 포인트

- FastAPI 앱 실행 구조 정리
- SQLAlchemy 모델, 세션, CRUD 분리
- CORS 설정으로 프론트 연동 가능하게 구성
- 입력값 검증 및 예외 처리
- 게시글 비밀번호 확인 로직 구현
- 챗봇 응답 로직에서 제공 JSON을 우선 사용
- 배포 환경(Render)에서 동작할 수 있도록 설정 파일 점검

## 7. 제출 관점에서 백엔드가 해야 할 일

- Git Repository URL 형태로 제출 가능해야 함
- `.env`가 제외된 상태여야 함
- 초기 데이터 포함된 `.db` 파일 제출 준비
- Render 배포 URL이 정상 동작해야 함
